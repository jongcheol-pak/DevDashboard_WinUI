# DevDashboard_WinUI 테스트 관리 기능 기획안

본 문서는 프로젝트의 각 로컬 항목들에 대해 "테스트 목록"을 관리할 수 있는 기능을 추가하기 위한 UI 및 아키텍처 기획안입니다. 선배님 피드백에 따라 코드를 배제하고 기능과 구조 위주로 서술합니다.

## 1. 개요
선배님께서 요청하신 "테스트 목록 작성, UI 표시, 항목 체크 및 진행 내용 추가"를 구현합니다. 
항목들이 너무 많아질 것을 대비해 **큰 카테고리(분류) 안에 세부 테스트(체크박스)들을 묶어 보여주는 중첩 구조**로 기획했습니다.

## 2. 데이터 구조 (Domain Layer)

### 2.1 새로운 단위 추가
테스트 관리는 두 가지 계층으로 나뉩니다.
- **테스트 카테고리 (TestCategory)**: 기능 단위의 큰 묶음을 뜻합니다. (예: "로그인 기능 테스트", "DB 연동 테스트")
- **테스트 항목 (TestItem)**: 해당 카테고리에 속한 개별 체크리스트입니다. 각각의 항목은 제목, **진행 내용(메모/로그 등)**, 체크 완료 상태, 완료 시간을 가집니다.

### 2.2 프로젝트와 연결
기존 대시보드의 각 프로젝트 항목은 여러 개의 '테스트 카테고리'를 가질 수 있도록 확장됩니다. 

### 2.3 데이터베이스
- SQLite에 위 두 모델을 저장하기 위한 테이블 2개(`TestCategories`, `TestItems`)가 영구 저장을 위해 추가되어야 합니다.

## 3. UI 구성 (Presentation Layer)

### 3.1 TestListDialog (테스트 팝업 뷰)
카테고리로 그룹화된 테스트 목록을 직관적으로 보여주며, 다음과 같은 구성을 가집니다.

- **필터 탭**: 상단에 진행 중 / 완료됨 / 전체 테스트를 볼 수 있는 탭 바 배치.
- **카테고리 추가부**: 최상단에 새 카테고리를 만드는 텍스트 입력창과 추가 버튼.
- **리스트 영역 (중첩 뷰)**:
  - **카테고리 헤더**: 굵은 텍스트로 카테고리명을 표시하며 카테고리 자체를 삭제할 수 있는 버튼 제공.
  - **카테고리 내부 영역**: 
    - 세부 테스트 항목들이 리스트 형태로 출력.
    - 리스트 각 줄마다 체크박스(통과/완료)와 테스트 이름 표시.
    - 해당 카테고리에 `새 항목 추가`용 텍스트 입력창 지원.
- **테스트 진행 내용 (메모) UI**: 
  - 세부 테스트를 클릭했을 때 아래로 내용이 펼쳐지거나(Expander), 추가 공간이 열려 진행 내용을 길게 확인할 수 있는 UI 구조를 채택.
  - 진행 내용을 그 자리에서 수정하고 저장할 수 있도록 아이콘 버튼 제공.

### 3.2 대시보드 카드 UI (DashboardView.xaml)
- 기존 각 카드의 하단 아이콘들 (Git, Todo, History) 옆에 **테스트 목록 버튼**을 배치하여 위 팝업을 띄울 수 있도록 연결합니다.
