## 2026-02-27 — [ToolTipService.ToolTip] x:Uid 패턴 전체 수정

### 수행한 작업 요약

`[ToolTipService.ToolTip]` x:Uid 패턴이 런타임 XamlParseException을 발생시키는 문제를 해결하기 위해 tooltip 설정 방식을 전면 수정

### 문제 원인

- resw에서 `ButtonName.[ToolTipService.ToolTip]` 키로 tooltip을 설정하는 x:Uid 패턴은 WinUI 3에서 런타임 XamlParseException을 발생시킴
- 첨부 속성(Attached Property)은 x:Uid를 통한 resw 자동 바인딩이 정상 동작하지 않음

### 해결 방법

1. **MainWindow.xaml**: 버튼에 `x:Name` 추가, x:Uid에서 tooltip 관련 키 분리
2. **MainWindow.xaml.cs**: `ApplyToolTips()` 메서드를 추가하여 `ToolTipService.SetToolTip()` + `LocalizationService.Get()` 조합으로 코드비하인드에서 다국어 tooltip 설정
3. **DashboardView.xaml**: DataTemplate 내부 버튼은 x:Uid 대신 직접 `ToolTipService.ToolTip="..."` XAML 속성으로 교체
4. **AppSettingsDialog.xaml**: IconDeleteBtn tooltip을 직접 XAML 속성으로 교체
5. **App.xaml.cs**: 진단용 UnhandledException 핸들러 제거 (디버깅 목적 코드 정리)

### 변경된 파일 목록

| 파일 | 변경 유형 | 내용 |
|------|----------|------|
| `MainWindow.xaml` | 수정 | 버튼에 x:Name 추가, tooltip x:Uid 제거 |
| `MainWindow.xaml.cs` | 수정 | `ApplyToolTips()` 메서드 추가, 진단 코드 제거 |
| `Views/DashboardView.xaml` | 수정 | tooltip x:Uid를 직접 XAML 속성으로 교체 |
| `Views/Dialogs/AppSettingsDialog.xaml` | 수정 | IconDeleteBtn tooltip을 직접 XAML 속성으로 교체 |
| `Strings/ko-KR/Resources.resw` | 수정 | ToolTip_ 접두사 키 7개 추가 |
| `Strings/en-US/Resources.resw` | 수정 | ToolTip_ 접두사 키 7개 추가 |
| `App.xaml.cs` | 수정 | 진단용 UnhandledException 핸들러 제거 |

### resw에 추가된 키
