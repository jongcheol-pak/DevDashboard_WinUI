## 2026-07 — Window.Title x:Uid XamlParseException 버그 수정

### 수행한 작업 요약

WinUI 3에서 `Window`에 `x:Uid`를 설정하고 `.resw`에 `XxxDialog.Title` 키가 있을 경우
`XamlParseException: Failed to assign to property 'Microsoft.UI.Xaml.Window.Title'`이 발생.
`Window.Title`은 DependencyProperty가 아닌 일반 CLR 속성이므로 x:Uid 할당 불가.
5개 Dialog에 동일한 패턴이 존재해 일괄 수정.

## 창 크기 변경 시 ContentDialog 크기 동기화

### 문제

창 크기를 조절해도 ContentDialog 패널 크기는 고정되어 있었음.

**원인**: WinUI 3 `ContentDialog`는 `XamlRoot` 변경 시 스크림(오버레이)만 자동으로 조절하고,
ContentDialog 패널의 `Width`/`Height`는 자동으로 갱신되지 않음.

### 해결

`DialogWindowHost.ShowAsync()`에서 `AppWindow.Changed` + `DidSizeChange`로 창 크기 변경을 감지하여
`XamlRoot.Size`(DIP 단위)를 `dialog.Width`/`dialog.Height`에 동기화:

```csharp
void SyncDialogSize()
{
    var size = host.Content.XamlRoot.Size;
    dialog.Width = size.Width;
    dialog.Height = size.Height;
}
SyncDialogSize(); // 초기 설정
host.AppWindow.Changed += OnAppWindowChanged; // 창 크기 변경 시 동기화
var result = await dialog.ShowAsync();
host.AppWindow.Changed -= OnAppWindowChanged; // 반드시 해제
```

- `XamlRoot.Size`는 DIP 단위 → DPI 변환 불필요
- `ShowAsync()` 완료 후 이벤트 핸들러 해제 필수 (메모리 누수 방지)

### 검증 결과

- 빌드: **성공** (에러 0개, 기존 MVVMTK0045/NETSDK1233 경고만)

## 재수정 — ContentDialog 패널 크기 동기화 (dialog.Width 방식 → BackgroundElement 직접 제어)

### 문제

`dialog.Width = size.Width; dialog.Height = size.Height;`를 설정해도 ContentDialog 패널이 고정 크기로 유지됨.

**원인**: WinUI 3 ContentDialog ControlTemplate 내부의 `BackgroundElement`(실제 패널 Border)가
`HorizontalAlignment="Center"`, `VerticalAlignment="Center"`로 하드코딩되어 있어서,
`dialog.Width`를 설정해도 패널 크기는 내용(Content) 크기에 맞게 고정됩니다.
추가로 `dialog.MaxWidth`(기본 548)가 `TemplateBinding`으로 `BackgroundElement.MaxWidth`에 전달됩니다.

### 해결

1. `dialog.MaxWidth = Infinity` → TemplateBinding 통해 BackgroundElement.MaxWidth 제한 해제
2. `ContentDialog.Opened` 이벤트에서 `VisualTreeHelper`로 `BackgroundElement`를 찾아:
   - `HorizontalAlignment = Stretch` → 창 전체 너비로 확장
   - `VerticalAlignment = Stretch` → 창 전체 높이로 확장
3. `AppWindow.Changed(DidSizeChange)`에서 `BackgroundElement.MaxWidth/MaxHeight`를 창 크기로 동기화

### 동일 실수 방지

- `dialog.Width/Height` 설정은 ContentDialog 패널 크기에 영향을 주지 않음
- BackgroundElement는 ContentDialog.Opened 이후에만 VisualTreeHelper로 찾을 수 있음 (그 전에는 Template 미인스턴스화)

## 창 최소 크기 미설정 — ContentDialog가 창 밖으로 잘림

### 문제

ContentDialog에 `MinWidth="680"`, `MinHeight="460"`이 설정되어 있어도
`DialogHostWindow` 창을 그보다 작게 줄이면 ContentDialog 패널이 창 밖으로 잘려나감.

**근본 원인**: `DialogHostWindow`에 최소 창 크기가 설정되어 있지 않아서
사용자가 창을 ContentDialog의 최소 크기보다 작게 조절할 수 있음.

### 해결

`ContentDialog.Opened` 이벤트에서 `BackgroundElement.ActualWidth/Height`(내용 기반 실제 DIP 크기)를
`WindowManager.Get(host).MinWidth/MinHeight`로 설정:

```csharp
// ① Stretch 변경 전에 읽어야 함 (Stretch 후에는 창 전체 크기가 반환됨)
var manager = WindowManager.Get(host);
manager.MinWidth  = backgroundElement.ActualWidth;   // DIP 단위
manager.MinHeight = backgroundElement.ActualHeight;  // DIP 단위

// ② 그 후 Stretch 설정
backgroundElement.HorizontalAlignment = HorizontalAlignment.Stretch;
backgroundElement.VerticalAlignment   = VerticalAlignment.Stretch;
```

### 동일 실수 방지

- `ActualWidth/Height` 읽는 순서: 반드시 `HorizontalAlignment = Stretch` **전**에 읽어야 함
- `WindowManager.MinWidth/Height`는 DIP 단위이므로 DPI 변환 불필요

## ContentDialog → Window 전환 (전면 교체)

### 수행한 작업 요약

### 핵심 패턴

**코드비하인드 공통 패턴:**

```csharp
public sealed partial class XxxDialog : Window
{
    private readonly TaskCompletionSource _closedTcs = new();

    public XxxDialog(...)
    {
        InitializeComponent();
        SystemBackdrop = new MicaBackdrop();
        var manager = WindowManager.Get(this);
        manager.MinWidth = MIN_W;
        manager.MinHeight = MIN_H;
        Closed += (_, _) => _closedTcs.TrySetResult();
    }

    internal Task ShowAsync()
    {
        DialogWindowHost.Show(this, InitW, InitH);
        return _closedTcs.Task;
    }
}
```

**호출부 변경 패턴:**

```csharp
// Before
var result = await DialogWindowHost.ShowAsync(dialog);
if (result == ContentDialogResult.Primary && dialog.ResultXxx is not null) { ... }

// After
await dialog.ShowAsync();
if (dialog.ResultXxx is not null) { ... }
```

### 빌드 이슈 및 조치

**CS1503 — Window를 FrameworkElement로 변환 불가 (6건)**

- 원인: WinUI 3에서 `Window`는 `FrameworkElement`가 아님. `{x:Bind Prop, Converter={StaticResource ...}}`를 Window XAML에 사용하면 x:Bind 코드 생성기가 `FrameworkElement` 타입을 요구하는 코드를 생성 → CS1503
- 영향 파일: CommandScriptDialog, AppSettingsDialog(2건), ProjectHistoryDialog, HistoryDialog, ProjectSettingsDialog
- 해결:
  1. DataTemplate 내부: `{x:Bind Prop, Converter=...}` → `{Binding Prop, Converter=...}` (DataContext = 아이템 자동)
  2. ViewModel 속성: 부모 StackPanel에 `DataContext="{x:Bind Vm}"` 추가 후 `{Binding Prop, Converter=...}`

### 검증 결과

- 빌드: **성공** — 경고 92개(MVVMTK0045 기존), **오류 0개**
- 테스트: 미수행
- 린트: 미수행

### 동일 실수 방지

- WinUI 3 `Window` XAML에서 `{x:Bind ..., Converter={StaticResource ...}}`는 사용 불가 → `{Binding ..., Converter=...}`로 대체
- DataTemplate 내부 converter는 `Binding`으로 변경해도 DataContext(=아이템)에서 자동 해결됨
- ViewModel 직접 참조 converter는 `DataContext="{x:Bind Vm}"` + `{Binding Prop, Converter=...}` 조합 사용
- `Opened` 이벤트(ContentDialog 전용) → `Grid.Loaded` + `_initialized` 가드로 교체
- `Hide()` (ContentDialog 전용) → `Close()` 로 교체
- `args.Cancel = true` (ContentDialog 검증 패턴) → `return`으로 교체 (OnSave 내)
